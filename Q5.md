## Question 5 - System Design Under 5GB VRAM

### Approach

The system is designed as a **dual-path architecture** to meet strict VRAM and real-time constraints:

* A **fast path** handles most frames using lightweight models and classical CV
* A **slow path** uses a VLM only when the fast path is uncertain

This ensures real-time performance without overloading the GPU.

---

## System Design

### 1. Fast System

This runs continuously and handles the majority of cases.

**Responsibilities**

* Pallet detection (YOLO)
* Tape detection
* Heuristic checks
* Fork pocket validation

**Key Properties**

* Runs at high frequency
* Low latency (<40 ms)
* No VLM usage

---

### 2. Slow System

This is triggered only when the fast system is uncertain.

**Responsibilities**

* Run VLM inference on cropped pallet image
* Perform reasoning
* Return structured decision (in JSON)

Uses: Moondream2

---

## Memory Allocation within the limit

| Component             | VRAM Usage |
| --------------------- | ---------- |
| YOLO + CV pipeline    | ~1.5 GB    |
| VLM (Moondream2)      | ~1.8 GB    |
| OS + Navigation stack | ~1.0 GB    |
| **Total**             | ~4.3 GB    |

---

## Optimization Techniques

### 1. Resolution Reduction

* YOLO runs on 640×480 instead of 1080p
* VLM receives only a cropped region 336×336

---

### 2. Conditional VLM Execution

* Triggered only when needed (~<5% cases)

---

### 3. Asynchronous Execution

* VLM runs in parallel thread

---

### 4. Model Optimization

* YOLO with TensorRT INT8 optimized
* VLM with 4-bit quantized

---

### 5. ROI-Based Processing

* Only relevant regions processed
* Reduces compute and memory load
