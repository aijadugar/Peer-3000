## Question 7 - New Warehouse Settings

### Approach

The system is designed to be **configuration-driven**, so adapting to a new warehouse does not require rebuilding the pipeline.

---

## 1. Config file

All environment-specific parameters are stored in a YAML/config file.

### Changs in config

* Tape color (blue to yellow)
* HSV thresholds for color detection
* Clearance margins
* Stability thresholds
* Camera and lighting parameters

### Example Config Change

```yaml id="cfg7a1"
tape_color: yellow
tape_hsv_lower: [20, 80, 80]
tape_hsv_upper: [35, 255, 255]

isolation_margin: 0.15
stability_offset_threshold: 0.12
```

---

## 2. Update to New Payloads

* Adjust thresholds in config

---

## 3. Lighting Adaptation

* Tune preprocessing parameters
* Update glare detection thresholds

---

## 4. Model Fine-Tuning

If the detector fails to recognize pallets reliably then

* Collect 50–100 labeled images
* Fine-tune YOLO

---

## 5. Prompt update for VLM

The VLM does not require retraining, only update prompt

```python id="cfg7a6"
prompt = base_prompt.replace("{tape_color}", config.tape_color)
```
