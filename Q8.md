## Question 8 - Evaluation Before Deployment

### Approach

The pipeline is evaluated as a **binary classification system (READY / NOT_READY)** with additional **safety and latency constraints**.
The goal is to prove the system is reliable enough for real-world deployment without causing unsafe pickups.

---

## 1. Classification Metrics

### Precision

* Measures how many predicted READY cases are actually safe
* High precision ensures **no wrong pickup**

**Code Logic**

```python id="m8p1a"
precision = TP / (TP + FP)
```

---

### Recall

* Measures how many actual READY pallets are correctly detected
* Lower recall is acceptable compared to precision

**Code Logic**

```python id="m8p2b"
recall = TP / (TP + FN)
```

---

### F1 Score

* Combined measure of precision and recall

** Code Logic**

```python id="m8p3c"
f1 = 2 * (precision * recall) / (precision + recall)
```

---

## 2. Safety Metrics

### False Pickup Rate

* Fraction of unsafe pallets incorrectly marked as READY

**Target**

```text id="m8p4d"
false_pickup_rate < 0.5%
```

---

### Unsafe Pickup

* System should never allow dangerous pickup

---

### Wrong Pallet Detection

* Robot should not pick unintended pallet

---

## 3. Latency Metrics

### Fast System (CV Pipeline)

* Real-time decision making

```text id="m8p7g"
latency_fast < 40 ms
```

---

### Slow System (VLM Fallback)

* Handles edge cases

```text id="m8p8h"
latency_slow < 700 ms
```

---

## 4. Evaluation Dataset

* Use labeled dataset from target warehouse
* Include:

  * READY cases
  * NOT_READY cases
  * Edge cases
