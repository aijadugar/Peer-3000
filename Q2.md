## Question 2 - Fork Pocket Clearance Detection

### Approach

Fork pocket clearance is a safety-critical step before the AMR attempts pickup. The system uses two complementary methods:

* Camera-based detection (always available)
* Depth-based validation (with sensor)

---

## Method 1 - Camera-Based Detection

The idea is to visually detect the fork pockets as dark rectangular regions at the base of the pallet.

### Steps

* Crop the bottom 25% region of the detected pallet
* Convert to grayscale
* Detect dark regions
* If pockets are not clearly detected then assume obstruction

### Code Logic

```python
x1, y1, x2, y2 = pallet_bbox

# Crop lower 25% of pallet
roi = frame[int(y2 - 0.25*(y2 - y1)) : y2, x1:x2]

gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)

# Threshold to find dark regions
_ , thresh = cv2.threshold(gray, 60, 255, cv2.THRESH_BINARY_INV)

# Count dark pixels
dark_ratio = np.sum(thresh > 0) / thresh.size

# Decision
pocket_clear = dark_ratio > 0.15   # enough dark area means pockets visible
```

---

## Method 2 - Depth Sensor Validation

This method verifies whether the pocket space is physically empty.

### Steps

* Map the pocket region into depth frame
* Measure distance
* Compare with expected pallet depth

### Code Logic

```python
depth_roi = depth_frame[int(y2 - 0.25*(y2 - y1)) : y2, x1:x2]

avg_depth = np.mean(depth_roi)

expected_depth = pallet_thickness + 0.02  # 2 cm margin

pocket_clear = avg_depth >= expected_depth
```

---

## Failure Handling Strategy

If the system cannot confidently detect fork pockets:

### Retry Logic

* Stop robot movement
* Slightly change viewpoint
* Re-run detection

### Escalation

* If detection fails twice then request human intervention

### Code Logic

```python
if pocket_status == "UNCERTAIN":
    retry_count += 1
    reposition_robot()

if retry_count >= 2:
    alert_operator()
```
